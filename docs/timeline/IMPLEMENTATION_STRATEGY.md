## Marker File Format

Each marker is a simple `.md` file with YAML front matter:

**Example: `_timeline/personal-journey/transition-journey.md`**

```yaml
---
id: transition-journey
date: 2021-08-01
title: "Transition Journey Begins"
description: "My journey as a transgender woman - exploring identity, growth, and authenticity."
icon: transgender
color: "#f84ef8"
timeline: personal-journey        # Parent timeline ID
has_breakout: true
breakout_timeline: transition     # Points to _timeline/transition/ folder
links:
  - title: "About My Journey"
    url: "/about/trans-journey.html"
    type: internal
tags:
  - transgender
  - personal
  - milestone
---

Optional markdown content here. You can write as much or as little as you want.
This content will be displayed in the content card when someone clicks the marker.

You can use **markdown formatting**, lists, links, etc.
```

**Example: `_timeline/bottom-surgery/decision.md`**

```yaml
---
id: bottom-surgery-decision
date: 2024-10-30
title: "Made the Decision"
description: "Decided to pursue gender confirmation surgery. Navigating the hurdles, preparation, and joy of this decision."
icon: check-circle
color: "#f84ef8"
timeline: bottom-surgery           # Parent timeline
tags:
  - transgender
  - bottom-surgery
  - milestone
links:
  - title: "Read: Bottom Surgery - Hurdles, Prep, and Joy"
    url: "/2024/10/30/bottom-surgery-hurdles-prep-and-joy.html"
    type: blog
---

This was a pivotal moment in my journey...
```

## Blog Post Integration

**In your blog posts, just add a timeline tag:**

```yaml
---
title: "Bottom Surgery: Hurdles, Prep, and Joy"
date: 2024-10-30
tags:
  - transgender
  - bottom-surgery        # Matches marker tags
  - personal
timeline_marker: bottom-surgery-decision  # Optional: direct link to marker
---
```

Jekyll will automatically:
1. Find all posts with matching tags
2. Display them in the marker's content card
3. No manual linking required!

## Implementation Steps

### Step 1: Configure Jekyll Collection

**In `_config.yml`:**

```yaml
collections:
  timeline:
    output: false
    permalink: /timeline/:path/
```

### Step 2: Create Folder Structure

```bash
mkdir -p _timeline/personal-journey
mkdir -p _timeline/transition
mkdir -p _timeline/coming-out
mkdir -p _timeline/bottom-surgery
mkdir -p _timeline/ffs
```

### Step 3: Create Marker Files

Create individual `.md` files for each marker (examples below).

### Step 4: Update Timeline Page

Modify `pages/personal-journey.html` to build JSON from Jekyll collection data.

### Step 5: Update JavaScript

Minimal changes - still reads JSON, but now it's generated by Jekyll from files.

## Benefits

### Easy to Add Content

**Want to add a new marker?** Just create a new file:

```bash
# Add "Started HRT" to transition timeline
nano _timeline/transition/started-hrt.md
```

```yaml
---
id: started-hrt
date: 2021-08-15
title: "Started HRT"
description: "Beginning hormone replacement therapy"
icon: prescription-bottle
timeline: transition
tags:
  - transgender
  - hrt
  - medical
---

Today I started hormone replacement therapy...
```

**That's it!** Jekyll automatically includes it.

### Easy to Add a New Timeline

**Want to add "Career Journey"?**

1. Create marker: `_timeline/personal-journey/career.md`
2. Create folder: `mkdir _timeline/career`
3. Add markers: `_timeline/career/first-job.md`, etc.

### Easy to Link Blog Posts

**Just match tags:**

```yaml
# In blog post
tags:
  - transgender
  - bottom-surgery
```

Automatically appears in all markers with matching tags!

## Migration from JSON

### Old Way (timeline.json):
```json
{
  "timeline": {
    "markers": [
      {
        "id": "marker-1",
        "title": "Event",
        "date": "2021-01-01",
        "hasBreakout": true,
        "breakoutTimeline": {
          "markers": [...]  // Nested forever
        }
      }
    ]
  }
}
```

### New Way:
```
_timeline/personal-journey/event.md  (5 lines)
_timeline/event/sub-event-1.md       (5 lines)
_timeline/event/sub-event-2.md       (5 lines)
```
